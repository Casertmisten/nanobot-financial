# 📄 PRD：基于 Nanobot 的个人金融智能分析助手（V1.0）

---

# 一、项目背景

随着金融信息碎片化程度加剧，用户在进行投资决策时需要同时处理：

* 新闻信息（实时且分散）
* 公司基本面信息（结构化数据）
* 行业与宏观分析（多来源）
* 历史研究报告（非结构化文本）

传统搜索引擎与单轮 LLM 问答无法提供**可追溯、结构化、多步推理的金融分析结果**。

因此，本项目基于 Nanobot 构建一个融合：

> **RAG + Deep Research + Web Search 的金融智能分析系统**

实现从“信息获取 → 多步分析 → 结构化输出”的完整闭环。

---

# 二、产品目标

## 🎯 核心目标

构建一个可用于以下三类任务的 AI 金融助手：

1. 金融知识问答（RAG增强）
2. 个股投资深度研究（Deep Research + Web Search）
3. 每日金融新闻总结生成

---

## 🎯 非目标（明确收敛范围）

* 不做实时交易系统
* 不做量化回测系统
* 不做多智能体协作框架
* 不做用户资产管理系统

# 四、功能需求设计

---

# ✅ 功能1：金融问答（RAG系统）

## 1.1 功能描述

用户输入金融相关问题，系统基于本地知识库（新闻、研报、金融文档）进行语义检索，并结合大模型生成**带引用的回答**。

---

## 1.2 输入

```text
用户问题（自然语言）
例如：
“苹果公司最近财报表现如何？”
```

---

## 1.3 处理流程

1. Query embedding
2. 向量数据库检索（Top-K）
3. 拼接上下文（Context augmentation）
4. LLM生成答案
5. 添加引用来源（citation）

---

## 1.4 输出

```text
- 结构化回答
- 引用来源（新闻/文档片段）
```

---

## 1.5 核心价值

* 提升回答可信度（可追溯）
* 降低 hallucination
* 建立金融知识底座

---

## 1.6 技术要求

* 向量数据库：FAISS / Chroma
* Embedding model：text-embedding / bge
* LLM：Qwen / GPT / DeepSeek
* Retrieval：top-k similarity search

---

# ✅ 功能2：个股深度研究（Deep Research + Web Search）

## 2.1 功能描述

针对“某只股票是否值得买入”的复杂问题，系统自动执行多步研究流程，包括：

* 问题拆解
* 多源信息检索（RAG + Web Search）
* 信息整合分析
* 结构化投资报告生成

---

## 2.2 输入

```text
“特斯拉现在是否适合买入？”
```

---

## 2.3 处理流程

### Step 1：问题拆解（LLM Planner）

拆解为子问题：

* 宏观经济环境
* 行业发展趋势
* 公司基本面
* 近期新闻影响
* 风险因素

---

### Step 2：多轮信息检索

#### 2.1 RAG检索

* 历史研报
* 新闻数据库

#### 2.2 Web Search

* 最新财经新闻
* 公司动态
* 市场舆情

---

### Step 3：信息整合（Reasoning）

LLM对多源信息进行：

* 归纳
* 对比
* 风险分析

---

### Step 4：报告生成

输出结构化分析报告。

---

## 2.4 输出格式（标准化）

```text
【投资分析报告】

1. 问题理解
2. 宏观环境分析
3. 行业趋势分析
4. 公司基本面分析
5. 近期事件影响
6. 风险因素
7. 综合结论（是否建议关注/买入）
```

---

## 2.5 核心价值

* 从“问答”升级为“AI分析师”
* 支持多步推理（Agent-like behavior）
* 融合外部实时信息

---

## 2.6 技术要求

* Multi-step LLM orchestration（Nanobot）
* Web search API（Serp / Bing / DuckDuckGo）
* RAG系统复用
* Prompt chaining / planning

---

# ✅ 功能3：每日金融新闻总结（Daily Briefing）

## 3.1 功能描述

系统每日自动收集多源金融新闻，并生成结构化市场总结报告，用于快速了解市场动态。

---

## 3.2 输入

```text
当日金融新闻数据（多源聚合）
```

来源包括：

* 财经新闻RSS
* 市场公告
* 行业资讯
* 热点事件

---

## 3.3 处理流程

1. 新闻聚合与清洗
2. 去重与分类（可选）
3. LLM摘要生成
4. 结构化输出

---

## 3.4 输出格式

```text
【今日金融市场总结】

1. 今日市场总体表现
2. 重要财经新闻
3. 个股/行业影响分析
4. 风险提示
```

---

## 3.5 核心价值

* 降低信息获取成本
* 提供结构化市场视图
* 可用于每日推送（未来扩展）

---

## 3.6 技术要求

* 新闻抓取（RSS / API / 爬虫）
* LLM summarization
* Topic clustering（可选简化）

---

# 五、系统设计约束（重要）

## 🚫 不做内容

* 高频交易策略
* 自动投资执行
* 用户资产管理系统
* 多agent复杂协作系统

---

## ✅ 设计原则

* **低复杂度优先**
* **pipeline清晰优先**
* **输出结构化优先**
* **可演示优先**

---

# 六、核心指标（用于面试表达）

## 功能指标

* RAG问答准确性（引用覆盖率）
* Deep Research 完整性（步骤覆盖率）
* 报告结构稳定性（格式一致性）

---

## 系统指标

* 单次 Deep Research 延迟 < 30s
* RAG检索响应 < 3s
* 日报生成可自动化运行
